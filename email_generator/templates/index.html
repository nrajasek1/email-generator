How it works

  1. Inputs validated for shape and length before they leave the browser.
  2. Core generator calls the model with retry on contract failures.
  3. Output passes through a strict schema — extra fields rejected.
  4. Final result returned identically across CLI, API, and this UI.

Guardrails

  • Plain text only. No HTML, no markdown.
  • Context-grounded. No invented pricing, names, or links.
  • Explicit failure. Bad model output is surfaced, not hidden.
